

# План исправления импорта юнит-экономики

## Обнаруженные проблемы

### 1. Неправильный маппинг колонок Excel

Текущий маппинг ведёт данные в неправильные поля базы данных:

| Колонка Excel | Текущий маппинг | Должен быть |
|---------------|-----------------|-------------|
| Реальная себестоимость, руб | `fabric_cost_total` | `unit_cost_real_rub` |
| Оптовая цена с учетом оптовой наценки, руб | `buyer_price_with_spp` | `wholesale_price_rub` |
| Розничная цена (оптовая+15%), руб | `retail_before_discount` | `retail_price_rub` |
| Маржинальность,% | `admin_overhead_pct` | `margin_pct` |
| Чистая прибыль, руб | `scenario_plan_profit` | `profit_per_unit` |

### 2. Опечатки в заголовках Excel

В файле Excel есть опечатки - колонки для ткани 2 и 3 имеют заголовок "Затраты ткани **1** на одно изделие, кг" вместо правильного номера. Это невозможно исправить через обычный маппинг заголовков.

### 3. Перезапись данных при автокалькуляции

Функция `calculateDerivedFields` всегда пересчитывает себестоимость, оптовую и розничную цены по формуле, даже если в Excel уже есть готовые значения.

---

## Решение

### Шаг 1: Исправить маппинг колонок (unitEconTypes.ts)

Исправить `excelColumnMap`:

```text
Удалить:
- 'реальная себестоимость, руб': 'fabric_cost_total'
- 'маржинальность': 'admin_overhead_pct'
- 'маржинальность,%': 'admin_overhead_pct'

Добавить:
- 'реальная себестоимость, руб': 'unit_cost_real_rub'
- 'себестоимость': 'unit_cost_real_rub'
- 'оптовая цена с учетом оптовой наценки, руб': 'wholesale_price_rub'
- 'розничная цена (оптовая+15%), руб': 'retail_price_rub'
- 'маржинальность,%': 'margin_pct'
- 'чистая прибыль, руб': 'profit_per_unit'
```

### Шаг 2: Добавить позиционный маппинг для опечаток (CostImport.tsx)

Вернуть позиционный маппинг ТОЛЬКО для колонок с известными опечатками:
- Колонка 15 (индекс) → `fabric2_kg_per_unit` (заголовок ошибочно "Затраты ткани 1...")
- Колонка 21 → `fabric3_kg_per_unit` (та же опечатка)
- Колонка 24 → `fabric3_cost_rub_per_unit` (заголовок "стоимость затрат ткани 2...")

### Шаг 3: Сохранять импортированные значения (useCosts.ts)

В функции `bulkUpsert` изменить логику - если в импорте уже есть `unit_cost_real_rub`, `wholesale_price_rub`, `retail_price_rub`, `margin_pct` или `profit_per_unit`, использовать их вместо автокалькуляции.

### Шаг 4: Расширить UnitEconFormData типы

Добавить в интерфейс `UnitEconFormData` недостающие поля для импорта:
- `unit_cost_real_rub`
- `wholesale_price_rub`
- `retail_price_rub`
- `margin_pct`
- `profit_per_unit`
