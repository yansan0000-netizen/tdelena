

# План: Принудительная очистка базы и исправление импорта

## Текущая ситуация

1. **В базе 2960 записей** — это накопленные данные от нескольких импортов
2. **Чекбокс "Удалить перед импортом"** был добавлен, но возможно не был активирован
3. **Виртуализация** — это просто оптимизация для быстрой работы с большими списками (отображаются только видимые строки). Это не проблема, а улучшение производительности.

---

## Решение

### Шаг 1: Очистить базу данных напрямую

Удалить ВСЕ записи из таблицы `unit_econ_inputs` (все пользователи) или только для текущего пользователя.

### Шаг 2: Включить чекбокс по умолчанию

Изменить `CostImport.tsx` — чекбокс "Удалить все существующие записи перед импортом" будет включен по умолчанию.

Изменение:
```typescript
// Было:
const [clearBeforeImport, setClearBeforeImport] = useState(false);

// Станет:
const [clearBeforeImport, setClearBeforeImport] = useState(true);
```

### Шаг 3: Добавить логирование для отладки

Добавить `console.log` в функцию удаления, чтобы видеть что происходит.

## Файлы для изменения

1. `src/components/costs/CostImport.tsx` — включить чекбокс по умолчанию

## Результат

- База будет очищена
- При следующем импорте останется ровно столько артикулов, сколько в Excel файле (~1755)
- Чекбокс "Удалить перед импортом" будет включен по умолчанию
