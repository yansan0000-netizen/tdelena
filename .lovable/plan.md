

# План: Исправление импорта юнит-экономики

## Обнаруженные проблемы

### 1. Накопление данных при повторных импортах
В базе **2960 записей** вместо ожидаемых ~1755. Причина — импорт использует `upsert`, который только добавляет/обновляет, но **не удаляет** старые записи, которых нет в новом файле.

### 2. Импорт несколько раз подряд
По логам видно два импорта:
- 03:52 — 2400 записей
- 03:53 — ещё 560 записей

Данные накапливаются между импортами.

### 3. Excel файл содержит ~6000 пустых строк
Файл имеет 7642 строки, но только ~1755 содержат реальные артикулы. Остальные — пустые с формулами `#DIV/0!`. Код корректно их пропускает, но проблема в накоплении.

---

## Решение

### Шаг 1: Добавить опцию "Удалить все записи перед импортом"

В компонент `CostImport.tsx` добавить чекбокс:

```text
[ ] Удалить все существующие записи перед импортом
```

При активации — перед импортом выполнять DELETE всех записей пользователя.

### Шаг 2: Создать функцию очистки в useCosts.ts

Добавить функцию `deleteAllCosts()`:
```typescript
const deleteAllCosts = async () => {
  if (!user) return false;
  
  const { error } = await supabase
    .from('unit_econ_inputs')
    .delete()
    .eq('user_id', user.id);
  
  return !error;
};
```

### Шаг 3: Очистить базу сейчас

Выполнить DELETE всех записей текущего пользователя.

### Шаг 4: Проверить, что артикул М319114Пи импортируется корректно

После очистки и повторного импорта проверить:
- Себестоимость: **259.82 ₽**
- Оптовая цена: **350.00 ₽**
- Маржинальность: **25.77%**
- Чистая прибыль: **90.18 ₽**
